## Architecture decisions (locked)

1. **Cloud-free, ever** — invariant 1 in [`architecture.md`](architecture.md). The single allowed network surface is the LLM/Whisper model download (Spec 19), allowlisted by file path.
2. **flutter_bloc over Provider.** Provider removed by Spec 08; package banned in Spec 10.
3. **Hive CE retained as KV store.** Typed adapters land in Spec 04b (deferred).
4. **P2P transport via `flutter_nearby_connections`** — the package is dormant (last update ~2 years). Spec 22 §A starts with a viability probe on iOS 17+ + Android 14+; if probe fails, the spec extends to write platform channels (`MultipeerConnectivity` + `Nearby Connections API`).
5. **STT via `speech_to_text`; TTS via `flutter_tts`** — both wrap native APIs with hard offline gate. Spec 15 adds an instrumented offline-trace test; if a device fails the trace, STT hides on that device. `sherpa_onnx` reserved as a future polish-spec fallback.
6. **On-device LLM via `llama_cpp_dart`** (replacing `fllama` per validation findings — `fllama` is at v0.0.1, ~17mo old, unverified). `flutter_llama` is the alternate; `MLC LLM` via custom FFI is the high-performance fallback. Final pick locks during Spec 18 benchmark.
7. **Whisper variant**: `whisper-base.en` for `AiTier.full`, `whisper-tiny.en` for `AiTier.compact` — confirmed by Spec 21 benchmark.
8. **Skills installed via `pnpm dlx skills`**; subagents from `VoltAgent/awesome-claude-code-subagents`. No invented skills, no invented agents.
9. **Specs drafted as a complete v1 roadmap; implementation is sequential.** Each spec has a `## Agents & skills` section that names the pre-coding skills and after-coding agents to invoke.
10. **Base theme: bone surface (`#EDE6D6`) + narrow-black ink (`#1C1B1A`)** — canonical default. Dark mode opt-in. 12 curated NotiThemeOverlay palettes (5 bone-first + 2 warm light + 5 dark) per Spec 11; per-note overlay fully replaces surface + accent + pattern + signature glyph. Default Notinotes accent: `#4A8A7F` (muted teal).
11. **Localization via `gen_l10n`** (built into Flutter SDK) per Spec 26 — replaces the earlier provisional `intl_utils` choice.
12. **`custom_lint` for the offline-imports rule** (Spec 02) — `analysis_server_plugin` is the modern alternative, considered future-direction when we accumulate ≥3 custom rules.
13. **Cross-spec auditor agents**: `architect-reviewer`, `security-auditor`, `performance-engineer`, `dependency-manager`, `refactoring-specialist` are dispatched at trigger specs (see [`AGENTS.md`](../AGENTS.md) "Custom agents") in addition to the per-spec named agents. Drift, invariant violations, and dependency creep get caught between specs.

15. **CLI-first, MCP-second**: Bash invocation of `dart`, `flutter`, `git`, `gh`, `xcrun simctl`, `adb` is preferred over wrapper MCPs. Only Playwright MCP is registered in [`.mcp.json`](../.mcp.json) — it's the single capability with no CLI equivalent. Rejected MCPs (`dart`, `git`, `ios-sim`, `mobile`, `memory`, `seqthink`, `github`) and their CLI replacements are listed in [`AGENTS.md`](../AGENTS.md).
16. **Offline invariant enforced by two layered gates** wired through Lefthook (Spec 02): pre-commit runs `scripts/check-offline.sh` (fast grep) and `dart run custom_lint` (AST-precise, fires the `forbidden_import` rule from `tools/forbidden_imports_lint/`). Forbidden patterns sourced from [`scripts/.forbidden-imports.txt`](../scripts/.forbidden-imports.txt) (single source of truth). Spec 02's draft prescribed a hand-rolled `scripts/install-hooks.sh`; superseded by the existing Lefthook setup, which already owned `.git/hooks/pre-commit` from the bootstrap commit. The previous `lefthook.yml` `pre-push.forbidden-imports` flat-regex grep was removed (redundant with the precise pre-commit gate). Two spec-text defects were corrected during implementation: (a) `flutter analyze` does **not** fire `custom_lint` v0.7 plugins from CLI — only IDE surfacing comes via `analyzer.plugins: [custom_lint]`; the CLI gate is `dart run custom_lint`. The rule now walks up from the analyzed file's path until it finds the patterns file (cached after first read).

Replaces the parallel-hierarchy plan (`blocs/`, `repositories/`, `screens/`, `widgets/<feature>/`) sketched in the bootstrap context docs. Provider-based files retained as `features/<feature>/legacy/<name>_provider.dart` until Specs 05+ migrate them to `flutter_bloc` and Spec 08 deletes the `legacy/` folders. No barrel files; cross-cutting and cross-feature imports use the absolute `package:noti_notes_app/...` form; intra-feature imports stay relative per [`code-standards.md`](code-standards.md). Spec 03's draft hardcoded "decision #11" but that integer was already taken by intervening decisions; recorded as #17. Both [`architecture.md`](architecture.md) and [`code-standards.md`](code-standards.md) had their layout trees rewritten in the same commit.
18. **Cross-cutting repositories at `lib/repositories/<resource>/`** when a domain resource is consumed by 2+ features (e.g. notes — read by home, note_editor, search, and the future share flow). Feature-private repos stay under `lib/features/<feature>/repository/`. Each repository is an abstract class with a sibling concrete impl (`notes_repository.dart` + `hive_notes_repository.dart`); methods return immutable models, never raw `Box`/`Map`. JSON-string-in-Hive storage retained until Spec 04b. Tests use a real Hive box on a temp dir (no mocking Hive); a `@visibleForTesting HiveNotesRepository.withBox(...)` constructor lets pure-Dart unit tests skip `Hive.initFlutter()` (which requires path_provider plugin channels). 20. **Per-route `NoteEditorBloc` with 22 events** (Spec 06): `BlocProvider.create` mounts the bloc when the editor route is pushed and disposes it when the route pops. Per-action persistence is preserved: every handler awaits `repository.save(note)` (or `repository.delete(id)`) before emitting state, mirroring the legacy `tooling*` central dispatcher's I/O profile. Side-effect signals (route pop on delete) ride on a one-shot `popRequested` boolean field reset to `false` by every subsequent `copyWith`; `BlocListener` reacts and `Navigator.of(ctx).pop()` is the only place `BuildContext` enters the picture — the BLoC itself never imports `BuildContext`. Sheets (style/reminder/tag) consume the same per-route BLoC instance via `BlocProvider<NoteEditorBloc>.value(value: bloc)` re-injection inside `showModalBottomSheet`'s builder. The `NoteEditorBloc.cancelNotification` callback uses a hash-based id (`note.id.hashCode & 0x7fffffff`) extracted to [`notification_id.dart`](../lib/features/note_editor/notification_id.dart) so editor-scheduled reminders match editor-deleted notification ids; `NotesListBloc` still uses index-based cancellation (open question 12 — pre-existing divergence not widened by Spec 06). Cross-note reads needed by the editor (e.g. tag sheet's most-used tags) read `NotesRepository` directly via `context.read<NotesRepository>()` rather than expanding the editor BLoC's scope. The legacy provider's 21 migrated mutation methods are annotated `@Deprecated('migrated to NoteEditorBloc; remove in Spec 08')`; `togglePin` and `deleteNote` carry the dual-ownership message because both `NotesListBloc` and `NoteEditorBloc` dispatch them.

21. **`SearchCubit` (UI-only Cubit) and `UserCubit` + `UserRepository` complete the BLoC migration except for the home `Notes` legacy husk** (Spec 07). Two repositories now exist: `lib/repositories/notes/` and `lib/repositories/user/`. `SearchCubit` is the canonical "trivial state-only" Cubit example — pure UI state machine with 7 mutation methods, no async, no streams. `UserCubit` owns the active user, profile-picture lifecycle, and the randomized greeting; the greeting pool ports verbatim from the legacy `randomGreetings()` and is fixed at load time (the home title displays the chosen pool entry directly, replacing the legacy `'$timeOfDay, $name'` concat). Single-record repository pattern is established: abstract interface + Hive impl + `@visibleForTesting withBox` ctor + paired `getCurrent()`/`watch()` API + on-disk image cleanup behind the interface. `HiveUserRepository.setPhoto` is mildly more strict than the legacy `updateProfilePicture` it replaces — it removes the previous file when the path changes (legacy never cleaned up). When a cubit owns a mutable domain model (here: legacy `User` without `copyWith`), the cubit clones the model into a new instance per mutation before emitting; reusing the same reference makes Equatable's `==` see no change and silently suppresses the emit (a latent issue in Spec 06's `NoteEditorBloc` that's only invisible because the editor screen rebuilds via local `TextEditingController`/`StatefulWidget` state, not via BLoC rebuilds). ## Open questions

0. **Spec 04b — pre-resolved design choices** (recorded 2026-05-04 during the Spec 04 session, before the spec body is drafted): (a) `Note` becomes fully immutable — all fields `final`, hand-written `copyWith` / `==` / `hashCode`; **no** `freezed` (Spec 05 explicitly defers freezed adoption). (b) Migration trigger: only when the legacy `notes_v2` box has at least one entry AND the new `notes_v3` box is empty — idempotent, near-zero cost on warm boots; the legacy box is deleted from disk after a successful migration. (c) Failure-mode for entries that fail to decode: quarantine to a sidecar `notes_v2_quarantine` box keyed by the original id with the raw JSON string preserved, then continue the migration; the migration reports the quarantine count to logs (no user-facing surfacing yet). (d) The `blocks` vs `content`/`todoList`/`imageFile` duality is **out of scope** — both representations carry through the migration unchanged; collapsing the duality is a future editor refactor.
